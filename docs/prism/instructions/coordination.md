# PRISM Coordination Instructions

Use this instruction set when the prompt is about plan readiness, actionable-node selection, handoffs, shared claims, multi-agent progress, or repo-wide execution flow.

## Role Focus

- Keep the shared coordination surface authoritative, current, and explicit.
- Prefer lightweight coordination facts over hidden session assumptions.
- Use coordination state to unlock work for execution agents, not to centralize every decision.
- Preserve warm execution chains when they improve throughput, but do not let continuity override urgent priority.
- Treat claims, artifacts, publish state, and handoffs as durable coordination facts that other agents must be able to trust.
