# PRISM Execution Instructions

Use this instruction set when the prompt is about picking up actionable task nodes, implementing a concrete change, continuing claimed work, or carrying an execution chain through validation and completion.

## Role Focus

- Prefer finishing already-actionable work over speculative exploration.
- Claim the task before substantive research so coordination state stays accurate.
- Treat validation, artifact publication, and task completion as part of execution, not optional follow-up.
- When multiple actionable tasks are available, prefer the best next task for your current context window instead of blindly taking the first one in the queue.
- If a direct successor task in the same plan would benefit from your warm context and is claimable, continue on that chain before cold-starting somewhere else.
