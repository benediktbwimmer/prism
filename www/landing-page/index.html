The Old Way: Amnesic Agents

Dumping massive text into a context window guarantees agents start from scratch every session. They repeat historical mistakes, misunderstand local architecture, and collide with each other when working asynchronously without central coordination.

✓

The PRISM Way: Full Context

A cold-start agent enters a PRISM repo and instantly recovers reality. It sees the code (Facts), hydrates historically learned understanding (Memories & Concepts), and coordinates via the current trajectory (Plans) without stepping on toes.

TypeScript as the Query Language

Forcing agents to guess intent using grep leads to loop failures. By embedding a JS/TS engine over the graph, PRISM provides an agent-native interface. Agents write compositional TS snippets to pull definitions, historical failures, and active execution plans in a single round-trip—closing the coordination gap.

// Discover unblocked work and relevant memory
const plan = prism.plan("plan:12");
const ready = prism.readyTasks(plan.id);
const context = prism.memory.recall({ 
  focus: ready[0].anchors 
});

return { ready, context };

The Three Pillars of Cognition

🔍

Spatial Perception (Facts)

Every file, symbol, structure, and git event mapped dynamically. Agents request precise semantic relationships instead of blind shell search to understand exactly what the repo encompasses.

🧠

Temporal Continuity (Knowledge)

Earned episodic and structural memories capture what working on the codebase has taught us. This learned knowledge is published back into .prism and rebound to live handles at runtime.

🎯

Execution Alignment (Intent)

First-class coordination via Plans. PRISM tells agents exactly what is being worked on, what is blocked, and what must be validated next, turning a smart repo into a full execution substrate.

📚

Repo-Native Vocabulary

Concept packs let PRISM speak your specific language. Instead of a generic taxonomy, PRISM binds your domain terms directly to their implementations, docs, and validation logic.

🤝

Conflict Avoidance

With tasks and claims, swarms of agents can work on the same repository asynchronously. They don't overwrite each other's work because PRISM manages the handoffs and blockages.

🔗

MCP Native

PRISM exposes its entire cognitive graph through the Model Context Protocol. It connects seamlessly to Antigravity, Cursor, Claude Desktop, and your custom agent infrastructure.
